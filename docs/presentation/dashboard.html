기획 — 정체성 정의

"공인중개사에게 필요한 걸 한 곳에서." 상품을 실물 vs 디지털·서비스로 나누는 핵심 규칙을 먼저 잡았습니다.

2

개발 — 프론트 + 백엔드

React로 화면을, Express·MongoDB로 회원·상품·장바구니·주문을, 포트원 V2로 결제를 직접 붙였습니다.

3

배포 — 인터넷에 올리기

DB(Atlas) → GitHub → 백엔드(Heroku) → 프론트(Vercel). 환경변수·빌드팩까지 직접 연결해 실제 도메인을 갖게 됐습니다.

4

검증 & 보안 — 끝까지

배포 후 숨어 있던 버그·보안 구멍을 찾아 고치고, 실서버에 직접 공격을 시도해 막혔는지까지 확인했습니다.

🔍

겉보기 ≠ 실제

로컬에선 멀쩡했지만 배포하니 드러난 것들 — 상품상세가 안 열리고, 회원 정보가 새고 있었습니다. 직접 확인해야 보입니다.

🔗

배포는 '연결'의 문제

코드보다, 환경변수·도메인·DB 접근 같은 조각들을 서로 잇는 것이 핵심이라는 걸 배웠습니다.

🛡

검증의 힘

클릭 테스트로 못 잡는 문제를 코드·실서버 점검으로 잡았습니다. 실제로 공격을 날려 '막혔다'를 확인했습니다.

🌱

비전공자 + AI = 끝까지

코드 한 줄 몰랐지만, AI와 함께 기획부터 배포·검증까지 완주했습니다.
