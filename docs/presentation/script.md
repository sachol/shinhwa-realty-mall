## 1. 자기소개 (60~75초)
> *아래는 초안입니다. 대표님 실제 경험·말투로 자유롭게 다듬어 주세요. `[ ]` 부분만 채우면 됩니다.*

"저는 **현장에서 일하는 공인중개사**입니다. 부동산 중개 실무를 하다 보면, 명함·도장·계약서식 같은 사무 준비물부터, 경매·교육·요즘은 AI 활용까지… **'이런 게 한곳에 정리돼 있으면 좋겠다'** 싶은 게 늘 있었어요.

그런데 그걸 **남한테 맡기는 게 아니라, 제가 직접** 만들어보고 싶었습니다. 코드는 전혀 몰랐지만요. 그래서 이번 **바이브코딩**을 시작하게 됐습니다. `[배우게 된 직접적 계기 한 줄 — 예: "강의를 우연히 보고 '나도 할 수 있겠다' 싶어서"]`

목표는 단순했어요. **공인중개사에게 진짜 필요한 것들을 한 곳에서 사는 쇼핑몰.** 그래서 만든 게 **신화 부동산 쇼핑몰 — 공인중개사 올인원 큐레이션 마켓**입니다."

## 2. 데모 — 데스크탑 (2~3분)
> 발표 5분 전 사이트를 한 번 열어 **서버 예열**해 두세요(콜드스타트 방지). 로그인된 상태로 시작 권장.

**① 메인 화면** `[홈 https://shinhwa-realty-mall.vercel.app]`
"좌측 상단에 보시면 **'신화 부동산 쇼핑몰 / 공인중개사를 위한 올인원 큐레이션 마켓'** — 이 사이트가 뭐 하는 곳인지 바로 보이게 했습니다."
"상품은 **4개 카테고리**로 나눴어요. **사무용품 · 교육콘텐츠 · 경매 컨설팅 · AI 과정·코칭** — 전부 중개사 현장에 필요한 것들입니다."

**② 카테고리 필터** `[카테고리 카드 클릭]`
"카테고리를 누르면 해당 상품만 걸러서 보여줍니다."

**③ 상품 상세 — '정체성' 포인트** `[상품 하나 클릭]`
"여기가 제가 신경 쓴 부분인데요. 상품을 **실물**과 **디지털·서비스**로 구분했습니다."
- 실물(명함·도장·현수막): "배송비가 붙고, **수량을 조절**할 수 있어요."
- 디지털(VOD·AI코칭): "배송이 없어 **무료**, 그리고 **수량은 1개 고정**입니다. VOD를 두 개 살 일은 없으니까요."

**④ 장바구니** `[장바구니 담기 → 🛒]`
"담으면 우측 상단 장바구니 숫자가 바뀌고, 장바구니에서 **수량·금액이 실시간**으로 계산됩니다. 실물이 있으면 배송비 3,000원이 자동으로 붙어요."

**⑤ 결제 → 주문완료** `[주문하기 → 포트원 결제창 → 카드결제]`
"결제는 **포트원**으로 실제 결제창이 뜹니다. (테스트 모드라 실제 청구는 안 돼요.) 결제가 **확인된 뒤에만** 주문이 만들어지도록 했습니다 — 유령 주문을 막으려고요."
`[주문 완료 화면]` "주문번호와 내역, 다음 단계 안내까지 나옵니다."

**⑥ 내 주문** `[내 주문 목록]`
"방금 주문이 내역에 바로 보입니다."

**⑦ 관리자 화면** `[admin 계정으로 /admin]`
"관리자로 들어오면 **대시보드**에 등록 상품·총 주문·매출이 한눈에 보이고요."
- `[주문 관리]` "주문 **상태를 제작중·배송중·완료로** 바꿀 수 있고,"
- `[상품 관리 → 삭제 클릭]` "상품 등록·수정·삭제가 됩니다. 삭제는 실수 방지로 **확인 모달**을 띄웠어요."

## 3. 데모 — 모바일 시연 (40~60초) ★강사 요구사항★
> **온라인 발표라 '크롬 반응형 모드'로 폰 화면을 보여주는 게 제일 깔끔합니다.**
> 미리 띄우는 법: 사이트에서 **F12 → Ctrl+Shift+M(📱 기기 툴바) → iPhone 12 Pro 선택**. 그 상태로 화면공유.
> (실제 폰으로 보여줘도 OK — 단 화면공유가 번거로우면 반응형 모드 권장)

`[크롬 반응형 모드(iPhone)로 전환 — 메인 → 카테고리 → 상품 → 장바구니 스크롤]`
"이번엔 **휴대폰 화면**으로 보겠습니다. (반응형 모드로 전환) 보시는 것처럼 로고·상품·장바구니가 **폰 크기에 맞게** 정리됩니다. 공인중개사분들이 **현장에서 폰으로 바로** 쓸 수 있게 만든 점이 포인트예요."
- 짚을 것: 상품이 **2열**로 깔끔하게 / 장바구니 수량·가격이 안 깨지고 / 로고도 작은 화면에 맞게 줄어듦.
